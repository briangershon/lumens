Component Guide

Lumens Starmap Banner

`@briangershon/lumens-starmap-banner` is an interactive custom element for adding a transparent animated sky with visible stars, constellations, and exotic unseen objects to a host layout.

@briangershon/lumens-starmap-banner
<lumens-starmap-banner>
Event: starmap-object-selected

Install

Install the package into any browser ESM application.

pnpm add @briangershon/lumens-starmap-banner
Import

Import the package once so the custom element registers.

import '@briangershon/lumens-starmap-banner';
Minimal Markup

Place the element anywhere inside a host-owned banner or header. 

<lumens-starmap-banner></lumens-starmap-banner>
Browser Bundle

If you are not using npm-based ESM imports, load the standalone bundle that ships with the package. 

<script
  type="module"
  src="./node_modules/@briangershon/lumens-starmap-banner/dist/lumens-starmap-banner.bundle.js"
></script>

<lumens-starmap-banner></lumens-starmap-banner>
Attributes

HTML configuration surface

Attribute Type Default Notes
speed number 900 Positive number controlling simulated time progression.
label-limit number 14 Positive integer limit for visible labels.
start-time string 2026-01-15T05:00:00.000Z ISO timestamp used as the initial sky state.
dark-mode boolean false Switches the foreground palette for darker host surfaces.
Properties

JavaScript-facing surface

Property Type Reflects Notes
speed number speed Matches the animation speed attribute.
labelLimit number label-limit Controls how many visible labels the banner shows.
startTime string start-time Accepts the initial sky time as an ISO timestamp string.
darkMode boolean dark-mode Enables the higher-contrast palette for dark hosts.
Events

Custom events emitted by the component

Event When it fires Detail shape
starmap-object-selected When the hovered astronomical object changes. { selected, id, name, type, observability, constellation, magnitude, distanceLy, altitude, azimuth }

When detail.selected is false, the rest of the payload fields are null. 

type StarmapSelectionDetail = {
  selected: boolean;
  id: string | null;
  name: string | null;
  type: string | null;
  observability: string | null;
  constellation: string | null;
  magnitude: number | null;
  distanceLy: number | null;
  altitude: number | null;
  azimuth: number | null;
};
Starter Example

Viable first integration

This is the recommended first example for a new site because it shows package import, host-owned layout, configuration attributes, and the emitted selection event together. 

import '@briangershon/lumens-starmap-banner';

const banner = document.querySelector('lumens-starmap-banner');

banner?.addEventListener('starmap-object-selected', (event) => {
  if (!event.detail.selected) return;
  console.log(event.detail.name, event.detail.type);
});
<header class="site-hero">
  <lumens-starmap-banner
    dark-mode
    speed="900"
    label-limit="14"
    start-time="2026-01-15T05:00:00Z"
  ></lumens-starmap-banner>
</header>
Behavior Notes

What to expect in a host page

Host-owned layout

The banner is intended to layer into a host header rather than take over the full page.

Light and dark surfaces

Use the default palette for light hosts or add dark-mode on darker surfaces.

Hover-driven detail

Host interfaces can react to the hover event to show supporting copy, labels, or analytics.
